## 1. 普通用户的标准安装方式（推荐）

对于非技术人员，您**绝对不需要**去安装 Node.js 或者学习任何命令行代码。请直接使用我们为您打包好的一键式图形安装包：

1. 用浏览器打开官方 Releases 下载页面：  
   👉 [GitHub Releases 下载页](https://github.com/2014-y/ClawAI/releases)  
2. 找到最新发布版，下载名为 `Nexora Agent Setup.exe` 的安装文件。  
3. 双击该安装包文件，按照弹出的安装向导指引一路点击「下一步」完成安装。  
4. 在桌面找到 **Nexora Agent** 的小机器人图标，双击打开即可开始使用。

> [!NOTE]
> 我们的安装包内已经内置了一套独立安全的 Node.js 绿色沙箱环境，即使是一台没有任何编程环境的全新 Windows 电脑，也能做到“双击即用”。

## 2. 硬件与环境要求

为了让 AI 助手稳定流畅地运行，您的电脑最好满足以下基本配置：

| 硬件项目 | 最低建议配置 | 备注说明 |
| :--- | :--- | :--- |
| **操作系统** | Windows 10 / 11 (64位系统) | 目前仅支持主流 Windows 64位桌面系统 |
| **电脑内存** | 4GB 及以上 (推荐 8GB) | 内存越大，多平台同时绑定时的运行越稳定 |
| **硬盘空间** | 至少预留 1GB ～ 2GB 可用空间 | 用于存放软件本身、大模型本地依赖和缓存数据 |
| **网络环境** | 顺畅且能正常访问公网 | 调用在线大模型（如 DeepSeek 等）时必须保持联网 |

> [!TIP]
> 如果您想玩一些更高级的“纯本地运行”玩法（比如配合本地 Ollama 模型），那么对您电脑的 CPU、显卡以及内存会有极高的要求。**对于普通用户，我们强烈建议您优先使用云端大模型的 API Key，不仅响应速度极快，还完全不吃电脑配置。**

## 3. 如何处理 Windows 系统的安全拦截提示

由于我们的桌面客户端尚未购买昂贵的微软官方开发者签名证书，因此在您双击安装包或首次打开软件时，Windows 防御系统（SmartScreen）可能会弹出警告弹窗。**这完全属于正常情况，请放心，软件本身无毒且完全开源。**

### 💡 解决步骤：
1. 当屏幕上弹窗提示 *“Windows 已保护你的电脑”* 或者 *“未知发布者”* 时：
2. 点击弹窗内容里不太显眼的 **「更多信息」** 文本链接。
3. 此时弹窗的右下角会多出一个 **「仍要运行」** 按钮，点击它即可继续正常的安装或启动流程。

### 🛡️ 杀毒软件误杀防护：
如果您电脑上安装了第三方杀毒软件（如 360安全卫士、火绒、腾讯电脑管家等），它们可能会在软件运行某些后台网关逻辑时进行拦截。若遇到提示，请务必选择 **“允许本次操作”** 或 **“加入白名单/信任区”**。

## 4. 安装成功后的“自检清单”

在您准备大展身手前，建议对照下表进行一次自检，全部打勾说明您的软件环境百分之百健康：

- [ ] **运行状态**：双击图标能顺利打开界面，且没有出现闪退。
- [ ] **网关状态**：点击「启动 Nexora Agent」按钮后，指示灯能成功变绿（🟢 已就绪）。
- [ ] **模型通路**：在「模型配置」里保存好 API Key 之后，进入「模型会话」发送消息能正常获得 AI 的回复。
- [ ] **渠道接入**：在「通讯管理」里成功扫码，且对应的微信卡片状态显示为“已绑定”。

## 5. 常见问题与排错方法（避坑指南）

### 5.1 双击安装包或者软件快捷方式完全没反应？
* **解决方法**：这通常是由于系统临时权限不足或被杀毒软件误杀导致的。请尝试右键点击安装包或桌面上的软件图标，选择 **「以管理员身份运行」**。如果还是打不开，建议暂时退出电脑上的杀毒软件，然后重新安装一遍。

### 5.2 软件能打开，但点击启动后指示灯一直显示红灯或卡在黄色？
* **原因分析**：这一般是因为上一次关闭软件时，后台的进程没有完全退干净，或者是系统的 18789 端口被占用了。
* **解决方法**：
  1. 彻底关闭 Nexora Agent 软件（记得右键点击右下角托盘图标，选择“退出”）。
  2. 键盘上同时按下快捷键 `Ctrl + Shift + Esc` 打开 **任务管理器**。
  3. 在进程列表中，仔细寻找名为 `node.exe` 或者 `Nexora Agent` 的多余进程，右键点击它们并选择 **「结束任务」**。
  4. 重新打开软件，再次点击启动，一般即可顺利变绿。

### 5.3 提示 18789 端口被占用？
* **原因分析**：Nexora Agent 后台的本地中转站需要使用 18789 端口。如果这个端口被别的软件抢走了，或者您之前开启的旧版程序没退干净，就会报这个错。
* **解决方法**：按上面 5.2 节的步骤，在任务管理器里把残留的 `node` 进程全部杀掉，然后重新打开软件即可。

### 5.4 指示灯已经变绿了，但是在聊天时 AI 完全不回答？
* **排查顺序**：
  1. **检查模型额度**：请去您购买或申请大模型的官网，确认您的账号没有欠费，且 API 额度没有过期。
  2. **检查 Key 格式**：请确认在「模型配置」里填写的 Key 没有带有多余的空格或换行，并且选用了正确的模型名字。
  3. **单体排查**：在左侧进入「模型会话」聊天。如果在软件里能聊，微信里不回，那说明是微信通道断开了，请去「通讯管理」重新扫码绑定。

## 6. 如何备份数据或迁移至新电脑

当您需要更换电脑，或者重装系统时，您一定不希望重新去申请 Key 和配置大模型。好在 Nexora Agent 的所有配置都保存在固定目录下。

您只需要备份并迁移以下两个文件夹（可在资源管理器地址栏直接输入并回车）：

| 备份内容 | 存放路径 | 作用说明 |
| :--- | :--- | :--- |
| **基础配置与记忆库** | `%USERPROFILE%\.openclaw` | 包含了您的模型 Key、微信配置以及 AI 助手的本地记忆文件（`MEMORY.md`） |
| **客户端本地数据** | `%LOCALAPPDATA%\NexoraAgent` | 包含了客户端桌面版的一些运行状态和本地缓存 |

> [!WARNING]
> **安全警示：**
> 上述文件夹中包含有您极其私密的大模型 API Key 和登录凭证，**请千万不要**将这些文件夹打包发送给任何陌生人，或上传至公网公开分享。

## 7. 面向开发者的源码安装与打包说明

> 💡 **提示**：本小节仅供具备 Node.js 基础的开发者阅读，普通用户请直接跳过。

### 7.1 本地开发环境准备
* **操作系统**：Windows 10 / 11 (64位)
* **Node.js 版本**：建议使用较新版本（推荐 **Node.js v24.x**）
* **依赖管理工具**：npm

### 7.2 获取源码与本地调试
在您电脑的终端中执行以下命令：

```bash
# 1. 克隆代码仓库
git clone https://github.com/2014-y/ClawAI.git
cd Nexora-Agent

# 2. 安装项目依赖
npm install

# 3. 运行本地开发环境，这会拉起 Electron 客户端外壳
npm run app:start
```

如果您只想单独调试底层的 Node 逻辑网关，而不拉起图形界面，可以直接运行仓库根目录下的：
```bash
node start-gateway.js
```

### 7.3 一键打包发布包
当您修改了代码并想发布给非技术用户使用时，可以通过以下命令打包出 Windows 下的绿色免安装目录与一键 Setup 安装程序：

```bash
npm run app:dist
```

打包完成后，所有的输出文件都会存放在根目录的 `dist/` 文件夹下。您只需要把 `dist/Nexora Agent Setup.exe` 分发给您的用户即可。
